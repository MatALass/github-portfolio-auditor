# Portfolio Decision Rules

## FEATURE_NOW
Use for repositories that are already strong enough to be shown publicly with minimal changes.

## KEEP_AND_IMPROVE
Use for repositories with solid foundations but visible weaknesses that should be fixed before featuring.

## MERGE_OR_REPOSITION
Use for redundant or poorly differentiated repositories that should be merged, reframed, or significantly repositioned.

## ARCHIVE_PUBLIC
Use for repositories worth keeping visible as history but not worth featuring.

## MAKE_PRIVATE
Use for low-signal repositories that damage overall portfolio quality.
